# Built-in Reference Viewport Contract v1

The `reference-viewport` module is Forge-owned, deterministic, data-only
projection for early P7 visual inspection. It accepts a strict bounded
`ReferenceScene`, validates inert identifiers, vertex/edge/frame cardinality,
coordinate limits, complete frame identity, and contiguous frame order, then
produces front, side, and top integer orthographic wireframes.

It has no filesystem, network, process, installer, package-manager, plugin,
script, URL, imported-project, renderer, GPU, runtime, engine, approval,
promotion, spending, credential, publishing, or protected-Kernel capability.
Unknown fields and executable-, path-, network-, or markup-shaped identifiers
fail closed. Forge Desktop may display the returned projection read-only using
its existing canvas; the frontend cannot reinterpret it as code.

A valid snapshot proves only deterministic projection of the exact typed scene
under `forge-wireframe-orthographic-v1`. It does not prove materials, lighting,
physics, production rendering, runtime compatibility, general player
preference, or final visual quality. Structured P7b observation receipts must
bind the exact scene fingerprint, viewport profile, frame, views, assertions,
and limitations before any human-review claim is retained.

Official owner-selected runtime software such as a future Unity installation
is not treated as hostile merely because it is external. Its later R1 adapter
still requires official provenance, pinned packages, a separate project,
backup/rollback, clean import, build, identifier, replay, and profiling proof.
Stronger containment is reserved for genuinely untrusted executable content:
unknown scripts, plugins, packages, native projects, or AI-generated code.
