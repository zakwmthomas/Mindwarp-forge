# G1 C3 Post-Interface Consumer Reassessment

Date: 2026-07-16

Status: **next design audit selected; no passage or downstream optical
implementation authorized.**

## Decision

The verified local interface event fits the master C3 route, but it does not
make C3 closure-ready. It supplies one exact observer-independent optical
interaction at a uniquely evidenced face. It deliberately does not construct a
new refracted path, prove endpoint transfer, establish biological visibility,
or answer whether an extended physical probe can pass through a volume.

The highest-information bounded next route is a **generic physical probe and
swept-volume passage mathematical design audit**. This is the previously
retained second consumer of the physical-path substrate. It should establish
the minimum observer-independent mechanical evidence C3 must own before any
implementation is considered.

This selection is an audit only. It does not select a probe shape, collision
engine, terrain model, organism body, navigation system, sphere, planet, biome
presentation or runtime adapter.

## Fit with the overall plan

`MASTER_PLAN_V2.md` explicitly assigns physical visibility and traversability
to C3, while C6 owns organism bodies, senses, locomotion meaning and ecological
occupancy. The completed bulk-transfer and interface-event references now give
C3 a bounded optical opportunity chain. The remaining generic-passage gap is a
separate named C3 obligation with no current mechanical substrate.

The route therefore preserves the intended causal order:

1. C3 records exact world-caused occupancy, clearance and contact opportunity.
2. C6 may later bind an organism-specific body and movement mode to that
   evidence without rewriting C3 truth.
3. A future runtime adapter may accelerate promoted queries, but cannot become
   their canonical definition.

Physical biome continuity is unchanged. Categorical occupancy, regions and
contacts cannot paint visible biome seams. Continuous causes still require
deterministic ecotone fades, and sharp transitions remain valid only where a
sharp physical cause is evidenced.

## Exact next design audit

The generic physical probe and swept-volume passage audit must compare narrow
candidates and record typed failure before choosing any implementation:

1. **Probe envelope.** Compare sphere, capsule, axis-aligned box and bounded
   convex candidates as query envelopes. A sphere here is only one local probe
   candidate; it is not permission to model a world or planet as a sphere.
2. **Motion scope.** Compare fixed-orientation translation with rotation and
   state explicitly whether v1 rejects rotation rather than approximating it.
3. **Geometry reconstruction.** Determine whether the current cell occupancy
   evidence can conservatively reconstruct a swept volume or whether a new
   exact substrate is required. A centre-line witness alone is insufficient.
4. **Initial overlap and boundaries.** Define closed typed results for starting
   penetration, tangent contact, simultaneous faces, unavailable cells,
   zero-length motion and outer-domain contact without epsilon nudges.
5. **Mechanical evidence.** Separate geometric clearance from substance- and
   face-bound blocking, penetration, support, friction or fluid-response
   evidence. Phase alone cannot imply passable, blocked or supportive.
6. **Output boundary.** Return physical clearance/contact/support evidence for
   one declared generic probe and motion. Do not emit a universal
   traversability score, route, gait, navigation result or destination fitness.
7. **Determinism and cost.** Bound arithmetic width, witness count, memory and
   work before code. Require hostile thin-barrier, corner, reversal,
   resolution-change and forged-contact counterexamples.
8. **Rollback.** Keep the existing point-path substrate unchanged. Any future
   passage module must be additive and removable without data migration.

## Failure points to engineer out

| Failure | Permanent response required by the audit |
|---|---|
| treat a point ray as clearance for an extended body | require a declared envelope and swept-volume evidence |
| infer `solid = blocked`, `liquid = passable` or `solid = support` | require separate typed mechanical interaction evidence |
| choose sphere because it is mathematically convenient | compare candidate envelopes against explicit use cases and counterexamples |
| turn a probe candidate into a planet/world-shape decision | keep the query envelope local and explicitly non-authoritative for world geometry |
| use one global traversability number | return reconstructable contact/clearance/support evidence only |
| import engine collision behavior as canonical truth | define engine-neutral semantics first; adapters remain later and replaceable |
| use floating epsilons or target-native layout | require checked deterministic arithmetic and target-neutral codecs |
| fork semantics for each platform | keep one portable core; fork an adapter only if measured PC/mobile or later console evidence requires it |
| let categorical cells create biome seams | preserve continuous causal ecotones and sharp-cause retention |

## Platform and ROI result

The proposed audit needs no platform-specific branch. A capability-free,
bounded integer reference can serve Windows PC and mobile from one semantic
core. Actual mobile execution, performance and memory measurements remain
required before promotion. PlayStation can be evaluated later through an
adapter against the same vectors. Xbox and Mac are lower-priority follow-on
targets; Linux is opportunistic. None of those outliers should force a fork or
block the design unless a priority target exposes a measured incompatibility.

## Stop condition

Run the generic physical probe and swept-volume passage mathematical design
audit above. Stop after selecting or rejecting a minimum candidate and naming
the required counterexample oracle. Do not add a schema, crate, dependency,
coefficient catalogue, collision engine, organism, navigation, terrain,
planet, biome presentation, runtime behavior, C3 closure or promotion without
a later bounded readiness route and applicable authority.
