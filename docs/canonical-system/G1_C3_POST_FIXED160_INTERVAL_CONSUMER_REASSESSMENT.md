# G1 / C3 post-fixed160 interval consumer reassessment

Date: 2026-07-16

Status: **consumer map complete; interval optical cell-step design audit
selected; no composer or source implementation authorized.**

## Decision

The fixed-160 conditional interval interface closes the local arithmetic half
of the prerequisite identified by the refracted-path composition readiness
correction. Given one exact oriented face, validated media and a declared
incident component box, it can now classify every RGB band and conservatively
enclose local reflected/transmitted power and direction.

It does not close the geometry half. The event does not carry a replayable
event-point enclosure, certify a next face, construct a positive-length
interval segment, preserve point-direction lineage, evaluate interval bulk
attenuation or prove endpoint arrival. A composer still cannot call the exact
Q32.32 point-path or bulk APIs after refraction without choosing a
representative ray.

The cheapest evidence-bearing next dependency is therefore an **interval
optical cell-step mathematical and oracle audit**. It must determine whether a
bounded point box plus one spectral direction box can certify one next cell
face and a containing next-point box for every admitted state. It stops before
bulk attenuation, interface evaluation, multi-step composition or schema work.

## Why the selected step comes before interval bulk

Bulk optical depth needs a physical length through one known medium. An
independent box of possible start points and directions does not imply one
exact segment, and pairing arbitrary endpoint corners can invent impossible
lengths. The cell-step must first prove that every admitted state exits through
one face and return an outward next-point enclosure. Only then can a separate
bulk audit bound all correlated lengths conservatively.

This ordering preserves one owner per operation:

1. cell geometry certifies the next physical boundary;
2. bulk optics bounds attenuation over the certified span;
3. interface optics bounds the local face event; and
4. a later composer may order their receipts without copying any kernel.

The next audit must decide whether interval cell geometry belongs as an
additive versioned surface in `physical-path-substrate` or in a separate
geometry owner. It may not bury geometry inside a composer merely to avoid the
ownership decision.

## Exact next design and oracle audit

The audit must remain capability-free and compare the surviving independent
axis-box candidate with any more correlated candidate only when the latter has
a fixed replayable cost. It must answer:

1. **Input evidence.** What minimum identities bind the validated volume,
   current cell, prior face, spectral band, source interval event, point box
   and direction box without nesting unbounded prior events?
2. **Point lineage.** Can the initial point box be derived from the exact
   point-path face event, and can every later point box replay from the prior
   cell-step receipt rather than a caller assertion?
3. **Universal face rule.** In three dimensions, can outward face-time
   intervals prove one strictly earliest positive face for every admitted
   point/direction pair? Overlap or equality must be
   `ambiguous_next_face`.
4. **Progress and boundaries.** Define zero-time prior-face re-entry,
   near-parallel positive motion, outer-domain exit, starting-on-face,
   unavailable neighbor and endpoint relation without epsilons or nudges.
5. **Output.** Return only a certified face, next cell and containing
   next-point/time boxes, or a typed ambiguity/failure. Do not return bulk
   transfer, another interface event, visibility or endpoint arrival.
6. **Arithmetic and cost.** Measure whether fixed checked 512-bit arithmetic,
   bounded decimal codecs and a fixed work ceiling cover the admitted domain.
   No native float or hidden higher-precision production decision is allowed.
7. **Utility.** Measure false-ambiguity and widening across ordinary, critical
   and repeated cases. Sound but nearly always ambiguous is not useful enough
   for implementation readiness.
8. **Rollback.** Preserve the existing exact path, bulk, point-interface,
   interval-interface and swept-AABB bytes, identities and dependencies.

The independent oracle must extend the existing exact-Fraction
counterexamples to full 3D and include one-Q1.62-unit face reversal, exact
edge/corner equality, correlation-erasure false ambiguity, prior-face zero
progress, near-parallel motion, sign reversal, outer-domain exit,
endpoint-on-face ambiguity, unavailable neighbor, overflow boundaries,
three-band dispersion and a 64-step widening/work portfolio.

## Platform and authority boundary

The audit targets one portable checked-integer semantic core for PC and mobile.
Windows x64/i686 execution and Android ARM64 compilation become later
implementation gates; actual mobile-device profiling remains a promotion
gate. No lower-priority platform justifies a semantic fork without measured
evidence.

Run the interval optical cell-step mathematical and oracle audit and stop after
selecting or rejecting a bounded representation, ownership location, typed
outcomes and candidate resource ceilings. Do not add a schema, crate,
dependency, interval bulk surface, composer, coefficient catalogue,
perception, rendering, runtime, organism, sphere, planet, terrain, biome
semantics, C3 closure or promotion without a later exact readiness route.
