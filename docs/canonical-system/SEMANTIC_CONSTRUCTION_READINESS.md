# Semantic Emergence to Construction Language: Readiness Package

**Status:** discovery and readiness only. This package does not generate assets,
choose a visual style, or implement geometry/runtime code.

## Purpose

The canonical path is causal rather than keyword-driven:

`world pressures -> roles -> competing solution families -> concepts -> capabilities -> typed part-role graph -> validated construction recipe`

The boundary prevents a semantic label from becoming untraceable geometry, and
prevents construction from inheriting hidden category-specific assumptions.

## Boundary to establish

| Record | Required role | Must not contain |
|---|---|---|
| `PressureContext` | Versioned world/history/gameplay constraints with evidence references | Free-standing personality/culture scalar claims |
| `RoleSet` | Explicit functional problems to solve | Geometry, asset category, or engine object |
| `SolutionFamilySet` | Multiple competing, bounded approaches with rejection rationale | One automatic “best” answer without comparison |
| `CapabilityGraph` | Causal capabilities and interfaces required by selected family | Vague words treated as implementation instructions |
| `PartRoleGraph` | Typed parts, relations, sockets, articulation/deformation/material roles | Renderer mesh, physics body, or raw executable code |
| `ConstructionRecipe` | Versioned graph operations and validator results | Ambient defaults, hidden editor state, promotion authority |
| `SemanticConstructionReceipt` | Explanation, alternatives, validation, costs, and limitations | Approval or player-facing quality claims without review |

## Core invariants

- Every construction-relevant claim traces through an explicit causal chain to
  a pressure, role, or declared gameplay requirement.
- A single label, synonym, or prompt wording cannot silently determine form.
- At least one rejected/alternative solution family is retained when choice is
  material, so diversity and rationale remain inspectable.
- Graph identity and recipe replay are independent of asset category and
  runtime/renderer objects.
- Typed sockets, support, collision, articulation, material regions, and
  interface constraints validate before later representation selection.
- Failure to explain, validate, or replay produces a failure receipt rather
  than a plausible-looking artifact.

## Paired fixture matrix

| Fixture | Required observation |
|---|---|
| Same PressureContext | Stable causal chain, alternatives, graph, and recipe fingerprint |
| Synonym/wording pair | Equivalent semantics do not change result merely through vocabulary |
| Poison-word input | Unsupported/descriptive language cannot bypass causal constraints |
| Contradictory pressures | Conflict is surfaced with alternatives/rejection; no silent incoherent merge |
| Diversity set | Distinct valid solution families remain distinguishable under the same broad role |
| Recipe replay | Same graph/recipe validates to equivalent construction record |
| Connectivity/support/socket/collision faults | Typed validator locates and classifies the failed relation |
| Version mismatch | Old semantic or graph contract is migrated/rejected explicitly |

## Required inspection

Reference Studio must display the causal chain, chosen/rejected alternatives,
capability graph, part-role graph, validator outcomes, recipe version,
fingerprint, limitations, and fixture/receipt links. It must not treat a visual
preview or a language-model explanation as proof of validity.

## Neighbour contracts

| Neighbour | Provides | Receives |
|---|---|---|
| Derived world rules/history | PressureContext and explicit mutable context | No ability to rewrite world/history canon |
| Representation selector | Validated PartRoleGraph and functional requirements | Representation decision record only |
| Asset factory/animation | Recipe, sockets, articulation/material roles, validation data | No category-specific hidden topology assumptions |
| ProofReceipt/Reference Studio | Inputs, alternatives, validator data, cost/limitations | Read-only evidence linkage only |

## Readiness gaps deliberately left open

The evidence does not select the pressure ontology, role vocabulary, comparison
scoring, diversity metric, graph type system, socket semantics, constraint
solver, recipe operation set, or semantic-equivalence policy. These choices
affect creative direction and long-lived content grammar; they must be designed
and reviewed as one bounded package, not inferred from AI output or prototype
examples.

## Entry criteria for a future implementation package

- World/history pressure schema and proof-receipt boundary are settled.
- A typed causal/graph contract names allowed values and explicit unknowns.
- Fixture matrix includes poison wording, synonym, contradiction, diversity,
  replay, and structural-invalid cases.
- Construction remains data-only, engine-neutral, and has no code execution,
  filesystem, network, or promotion authority.
