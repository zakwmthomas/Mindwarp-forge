# Recovered Source Audit

This is a provenance map, not a claim that every recovered prototype is
production-ready.

## Canonical design evidence

- `FORGE_MASTER_SPEC.md`: accepted architecture for the field matrix, lazy
  hierarchy, construction language, significance, scheduling, studio, testing,
  and controlled self-hosting.
- `FORGE_STATUS.json`, `CURRENT_STATE.md`, and `COMPILER_STATUS.md`: bounded
  status claims and explicit open work.
- `HOW_ZAC_AND_AI_WORK.md` and `TESTING_METHOD.md`: operating doctrine and
  evidence requirements.

## Evidence rule

Each package is classified as source evidence until its relevant test can be
re-run, its inputs/artifacts can be inspected, and its result is recorded in a
new engine-neutral proof pack. Python timings, hand-authored visual rigs, and
old reports do not establish runtime-engine performance or end-to-end asset
generation.
