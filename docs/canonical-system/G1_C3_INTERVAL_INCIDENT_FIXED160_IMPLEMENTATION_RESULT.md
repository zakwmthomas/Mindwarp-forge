`visible-radiance-interface-event` owner. A private `interval` module adds

## Compatibility and measured cost

Five point-v1 public cases lock canonical input bytes, event bytes, input IDs
and event IDs across normal incidence, index match, reverse direction,
critical total internal reflection and unsupported model outcomes. All remain
byte-identical after the additive module.

The widest focused three-band fixture measured 1,741 input bytes and 2,803
event bytes. Decoded Rust structure sizes were 608 bytes for the interval input
and 1,528 bytes for the event, below their codec ceilings. One local event
performs at most three 160-bit evaluations; a hostile continuation portfolio
does not inflate the local receipt.

The readiness record requested a public byte/ID lock for the retained
wide-coprime case. That case exercises a private arithmetic kernel and has no
public input/event encoding. Inventing a public encoding would have widened
the contract solely for a test. The durable correction is the existing private
kernel regression plus the independent exact fixed-portfolio checksum, while
the public critical-TIR case is included in the point-v1 byte/ID lock.
