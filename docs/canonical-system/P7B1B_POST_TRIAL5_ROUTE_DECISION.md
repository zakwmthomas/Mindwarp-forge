# P7b-1b Post-Trial-5 Route Decision

## Decision required

Trial 5 produced no debug event and its original cleanup was not self-contained:
one exact staging directory required verified host cleanup after runner exit.
Although the source now has a concrete ordering and detach repair, executing it
again is no longer a routine green test under the standing delegation. It has a
demonstrated cleanup regression and consumes another LPAC profile/process run.

The canonical route therefore has two honest choices.

## Option A — one repaired observer validation (recommended)

Authorize exactly one new run identity and fresh receipt using the repaired
existing broker and exact retained static candidate. Before execution it must:

- reproduce a locked/offline runner build and bind its new hash;
- retain the exact static candidate hash;
- pass eight runner tests, fixed-command negative fixtures, focused P7, full
  Forge, medium-integrity, and zero-residue preflight;
- preserve zero capabilities, LPAC, job, mitigations, child restriction,
  immutable hashes, three-second wall bound, ACL restoration, profile deletion,
  diagnostic-only claims, and independent cleanup checks;
- permit no external debugger program, dump, memory/register/stack/symbol read,
  registry capability, containment weakening, or second retry.

Expected gain is high relative to one bounded run: it can validate the repaired
event ordering and may finally retain DLL-load/exception/exit order. Uncertainty
remains high that the trace names the failing DLL. If cleanup is not automatic,
the run is immediately failed, exact owned residue is removed only after path
verification, and no further execution is permitted.

## Option B — leave P7b-1b blocked

Do not execute again. Retain the four clean failed trials, the static proof,
the failed/independently-cleaned Trial 5, and the unexecuted prospective repair.
This is the lowest-risk choice but keeps the P7b containment-denial proof red,
prevents P7b-1c renderer compatibility from inheriting a verified boundary,
and leaves F5 incomplete; G1 and R1 remain gated.

## Rejected silent routes

- A Trial-5 retry under the same receipt or identity is forbidden.
- A new canary/runtime, external debugger, trace provider, dump, capability,
  weaker AppContainer mode, removed mitigation/job shield, or different host is
  a different program or project approach and needs its own design decision.
- Advancing P7b-1c, completing F5, or opening G1 while treating denial as
  proved would silently skip a hard evidence gate.

## P10 stop rule

Baseline is zero retained events and one manual exact-stage cleanup. Option A
gets exactly one repaired validation. Success means an automatically cleaned,
claim-limited event receipt—not a denial pass. Failure, zero events, or any
cleanup residue ends this diagnostic family. No optimization or retry follows.

## Exact owner response

- Reply **“Run the repaired observer once”** to select Option A.
- Reply **“Leave P7b-1b blocked”** to select Option B.

The standing routine-test delegation remains active for ordinary bounded tests;
this choice is requested only because Trial 5 demonstrated cleanup failure.
