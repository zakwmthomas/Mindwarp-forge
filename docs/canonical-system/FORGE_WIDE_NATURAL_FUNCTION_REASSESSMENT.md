# Forge-wide Natural-function Reassessment

**Status:** owner-authorized F5 planning interruption, completed audit. This
record changes the plan and proof policy; it does not apply a solver to Forge
operations or game generation.

## North star and decision

Mind Warp remains a seeded, causally generated, inspectable universe whose
canonical state is compact and reconstructable, while Forge remains the
evidence, continuity, testing, and bounded-control system used to build it.
Mobile-to-high-end fidelity, organism diversity and growth, selective tracked
aging, visual quality, owner taste, recoverability, and strict authority
separation remain first-class constraints.

Nature-inspired mechanisms are adopted as a **tested method library**, not as a
new universal engine or a truth claim. The reusable object is the protocol:
state a target, expose hard constraints and trade-offs, probe cheaply, compare
with a simpler baseline, retain the trace and counterexample, and freeze only a
candidate. Each domain owns its meanings, objective, heuristic, fixtures,
budget, and promotion boundary.

The audit found one real architectural delta worth testing: bounded
goal-grounded search. Everything else is either an existing Forge mechanism, a
domain-local algorithm candidate, a deferred product choice, or an unsupported
claim.

## Philosophy compatibility

The protocol is compatible with Forge only when all of the following remain
true:

1. Evidence is not authority; imported metaphors and generated traces are
   candidates only.
2. Canonical facts remain typed, versioned, replayable and provenance-linked.
3. Objectives and metrics remain local; there is no universal fitness score.
4. A simple baseline, counterexample, cost and non-applicable scope are kept.
5. Cheap simulations precede expensive execution without replacing final
   integration evidence.
6. Visual output is inspected at useful scale, with uncertainty routed to a
   bounded owner check.
7. Failure is useful evidence. A natural-looking result is not correctness.
8. Runtime, networking and commerce remain gated behind their own architecture,
   security, performance and authority decisions.

## End-to-end traces

### Semantic construction pilot

`pressure evidence -> grounded roles -> hard acceptance envelope -> competing
mechanism families -> bounded proposal probes -> local validator -> trade
comparison -> candidate trace -> existing recipe proof`

The grounded protocol may change proposal order. It may not invent evidence,
replace diversity validation, or select a production artifact.

### Forge diagnosis pilot

`observed failure -> bounded hypotheses -> cheap discriminating probes ->
retained results -> revised ranking -> diagnosis candidate -> normal verified
repair workflow`

The harness includes the recurring retained Windows defects: an open SQLite
handle and incomplete portable absolute-path rejection. Those facts seed a
local diagnostic adapter; they do not justify a universal ranking.

### Selective aging trace

`canonical biological age -> eligibility -> per-save adult-appearance lock ->
presentation age -> morphology/material/animation adapters`

Age always advances for relationship-tracked eligible entities. Before adult
age, the lock has no effect. At or after adult age it presents adult form. When
disabled, the current biological stage is revealed. Old age never causes death.
Ambient entities use seeded age diversity rather than continuous lifetime
simulation.

## Reconciled execution route

1. **N0 complete:** inventory, redundancy, philosophy, failure and dependency
   audit in this record.
2. **N1 complete:** capability-free deterministic harness and case portfolio in
   `grounded-search-protocol`.
3. **N2 complete:** approve only the scoped natural-method policy and record
   negative transfer; no operational adapter is activated.
4. **N3 evidence gate:** run focused, modularity and full Forge verification,
   present results in chat, and retain the no-application boundary.
5. **Resume H5:** one large phone-legible reference/stage per image; continue
   actual-pixel human-reference intake and articulated visual comparison.
6. **Later organism route:** generalize morphology, coordinated fidelity,
   growth, ambient age distribution and selective tracked aging in dependency
   order. Do not silently enlarge the H5 humanoid proof.

## Stop and rollback rules

- Stop a method when it fails a hard constraint, costs more without verified
  gain, loses to its baseline, masks a local regression, or has no reliable
  target resistance estimate.
- Classify results as case-local, domain-local, cross-domain candidate,
  universal-protocol candidate, rejected, or inconclusive.
- Three different successes plus an explicit failure boundary may support reuse
  of the **protocol**. They never prove one heuristic, parameter set or natural
  mechanism universal.
- Remove the proof crate and policy references to roll back this reassessment;
  no production behavior or canonical game state depends on them.
