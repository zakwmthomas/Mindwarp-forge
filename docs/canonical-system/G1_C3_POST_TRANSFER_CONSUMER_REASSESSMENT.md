# G1 C3 Post-Transfer Consumer Reassessment

Date: 2026-07-16

Status: **next design package selected; no interface or passage implementation
authorized.**

## Decision

The verified bulk-transfer result does not make generic passage the next safe
package. The smallest dependency-safe next step is a **visible-radiance
interface-event mathematical design audit**. It must define one typed local
event calculation for a uniquely evidenced interface and stop before composing
that event into end-to-end path transfer.

Generic probe/contact-and-passage remains the following separate design route.
It still needs a declared probe envelope, a swept-volume substrate, initial
overlap rules, mechanical interaction evidence and support/contact semantics.
Those are more missing primitives than the local optical event requires.

C3 remains open. This reassessment selects a design question, not a schema,
coefficient catalogue, implementation, readiness claim or promotion.

## Why the current straight-path transfer cannot simply gain a multiplier

The existing bulk reference evaluates one declared straight segment. At a
refractive interface, the transmitted direction can differ from the incident
direction. Continuing along the original witness after that event would then
evaluate the wrong cells and can falsely claim transfer to the declared
endpoint.

Therefore the next package must reject the tempting formula:

`end_to_end = bulk_before * interface_transmission * bulk_after`

unless a later proof establishes that the transmitted direction remains on the
declared path, such as an explicitly admitted normal-incidence or index-matched
case. General refractive continuation needs a new path after every event and a
separate bounded propagation/path-composition design.

## Exact next design audit

The interface-event mathematical audit must compare a deliberately narrow
smooth-dielectric candidate against typed rejection and determine all of the
following before any readiness document:

1. **Event reconstruction.** Derive a candidate event only where the exact
   witness proves a transition between distinct positive-length media at one
   unique cell face and one exact path parameter.
2. **Geometry authority.** Decide whether an axis-aligned cell face is only a
   discretization boundary or sufficient declared interface evidence. Never
   infer a physical normal from substance identity alone.
3. **Ambiguity.** Edge, vertex, simultaneous, coincident-lane, tangent,
   stationary and unavailable cases require closed typed outcomes rather than
   axis priority or epsilon nudging.
4. **Interaction ownership.** Separate per-medium refractive evidence from
   boundary-specific coating, roughness or surface evidence. A universal
   substance property bag is forbidden.
5. **Admitted model.** Compare a smooth, lossless, unpolarized dielectric event
   with explicit unsupported outcomes for conductors, absorption at the
   boundary, roughness, scattering, fluorescence, polarization and undeclared
   evidence.
6. **Deterministic arithmetic.** Bound incident direction, oriented normal,
   index ratio, Fresnel reflectance/transmittance, refracted direction and total
   internal reflection without floats, epsilons, clamps or falsely exact
   scalars.
7. **Output boundary.** Return local event evidence only. Do not claim the
   refracted ray reaches the original endpoint, do not multiply it into the
   current bulk result, and do not infer perception or rendering.
8. **Counterexample proof.** Use a disposable high-precision oracle before a
   schema or implementation-readiness audit. Required cases include
   index-matched media, normal incidence, grazing incidence, both travel
   directions, critical angle, total internal reflection, equal-medium cell
   subdivision, face versus edge/vertex crossings and forged normals.

## Failure points engineered out

| Failure | Permanent design response |
|---|---|
| reuse coarse surface reflectance as interface transmission | reject: it lacks path-substance identity, incidence and interface provenance |
| infer `solid = dielectric` or phase-based refractive index | reject: phase supplies no interface class or coefficient |
| treat every cell boundary as a physical surface | merge equal-medium subdivision and require explicit unique interface evidence |
| continue the original segment after refraction | return a local event; later path composition must rebuild the changed path |
| choose an axis at edge or vertex crossings | return typed ambiguous geometry |
| silently treat missing evidence as transparent or opaque | return typed unavailable or unsupported evidence |
| use `T = 1 - R` for every material | admit it only inside a proven lossless dielectric model; other models remain unsupported |
| mix optical and mechanical material behavior | keep separate profiles keyed through shared identities only |
| convert optical transfer into biological visibility | perception, receiver sensitivity and detectability remain consumer-owned |
| let categorical regions paint biome seams | continuous causes still produce ecotones; sharp seams require sharp physical causes |

## Why generic passage remains later

Authoritative collision-query practice treats passage-relevant questions as
shape sweeps/casts, not point rays. Supported shapes, initial overlap,
translation versus rotation, impact normal, contact point, closest versus all
hits and numerical limitations are explicit parts of those APIs. Forge has not
selected any of those semantics, and importing a runtime physics engine would
violate the capability-free C3 boundary.

The later probe audit must therefore compare at least sphere, capsule, box and
bounded convex envelopes; fixed-orientation translation versus rotational
sweeps; initial-overlap and unavailable evidence; closest-hit versus complete
contact evidence; and clearance/contact evidence versus support or
traversability conclusions. It must still stop before organism bodies, gait,
navigation, destination fitness or runtime collision authority.

## Exact next action and stop condition

Run the visible-radiance interface-event mathematical design audit above. Stop
after selecting or rejecting a bounded mathematical candidate and naming the
required oracle. Do not implement an interface module, extend bulk transfer,
choose real coefficients, start the generic-probe package, add perception or
rendering, select a sphere/planet/terrain/runtime, close C3 or promote the
existing prototype.

Biome continuity is unchanged: categorical cells, physical regions and
interface identities cannot paint visible biome seams. Continuous physical
causes must produce deterministic fades, and a sharp transition is valid only
where a sharp physical cause is evidenced.
