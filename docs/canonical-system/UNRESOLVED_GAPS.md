# Unresolved Gaps and Explicit Gates

## Provenance and context

- The original four-day conversation is not available as a complete raw
  transcript. The recovered survival pack is high-value source evidence, not a
  verbatim replacement.
- Phone-originated capture must surface a source-gap receipt when local evidence
  is incomplete; Forge must never silently assume full capture.
- A long real-corpus compiler fixture and versioned interpretation migration
  rules are still required.

## Forge execution

- Current controlled application is not a trusted arbitrary-code sandbox.
- The historical Windows path escape means no prototype sandbox may be trusted
  until hostile path, symlink, environment, network, process, crash, and
  rollback tests pass.
- External AI and research connectors remain optional, isolated, and require
  separate approval for credentials, cost, or installation.

## Game-canonical production

- Field, scheduler, significance, representation, and semantic prototypes need
  engine-neutral reference proof before production status.
- Derived physics, history, construction language, asset factory, and animation
  remain specified or partial rather than finished production systems.
- Visual recognisability, player value, and hardware cost require structured
  review; numerical success alone is insufficient.

## Runtime

- No game engine is selected.
- No Unity, Godot, or custom runtime project is created or modified.
- Runtime selection is gated on the engine-neutral systems and the scored
  runtime-adapter decision package.
