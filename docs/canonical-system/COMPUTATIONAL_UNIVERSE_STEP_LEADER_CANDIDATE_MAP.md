## Intake and correction

The owner supplied a Gemini conversation titled *Universe as a Computational
System* and a workflow model: mainline implementation acts as the lightning
channel, while bounded step leaders explore nearby alternatives and reconnect
only when they improve the route to the objective.

The source is exploratory conversation, not scientific or architectural
authority. Its mechanisms are separated from its claims before transfer.

| Source idea | Disposition | Engineering form |
|---|---|---|
| Simple rules can create complex outcomes | Useful candidate | Versioned local rules plus deterministic emergence and explicit validity tests |
| Unstable candidates disappear | Useful only inside bounded experiments | Generate alternatives, test against local constraints, retain rejection evidence |
| Space and time should use different detail | Strong existing fit | Shared significance evidence with independent spatial, temporal, simulation and representation fidelity |
| Unobserved reality costs zero bytes | False as a universal rule | Unmaterialized canonical state may be derivable, but recipes, identity, history, deltas and provenance still cost storage |
| Natural changes can always be discarded | False as a universal rule | Discard only explicitly reconstructible non-authoritative state; world consequences and player-visible history may be canonical deltas |
| No active global ticks | Overbroad | Prefer event/lazy/multirate work where equivalent; retain ticking where causal fidelity or bounded integration requires it |
| Observer arrival creates truth | Rejected | Observation may trigger materialization or presentation; canonical identity and physical truth remain observer-independent |
| Effective-field theory proves LOD | Analogy only | A local reduced model needs error, validity-domain and cost evidence |
| Fixed 100/300/500 metre rings | Unproved parameter proposal | Derive fetch/process/render horizons from latency, velocity, cost and platform measurements |
| P2P DHT, replication eight and blockchain cold storage | Deferred architecture candidates | Require threat model, consistency, abuse, privacy, availability, platform and operating-cost evidence after runtime/network selection |
| Lowest timestamp wins conflicts | Rejected default | Conflict semantics must preserve causality, authority, idempotency and domain invariants |
| Lightning propagation is universally optimal | Metaphor, not proof | Use bounded value-of-information search with an explicit mainline, budget and stop rule |

## Reusable mathematical protocol

`mainline -> trigger -> bounded divergence -> complete map -> cheap falsifier -> local experiment -> reconnect/reject/defer -> resume mainline`

For each target-local assessment:

`VOI = sum(P(o) * best_local_utility_after(o)) - current_best_utility - probe_cost`

`LocalNetGain = closure_gain + quality_gain + future_cost_saved - implementation_cost - migration_cost - recurring_cost - complexity_risk_cost`

The shared controller ranks only strictly positive local values. It does not
compare unlike utilities, average systems, create domain meaning or apply a
result. A regression in one participating target blocks reuse regardless of
portfolio gain.

## Ranked transfer route

1. **Immediate local proof:** Forge research plus control-plane trigger and
   reconnection fixtures.
2. **Second independent proof:** context-compiler repeated-correction intake.
3. **First game-local trials after dependencies:** lazy hierarchy/history and
   significance/streaming using their existing local metrics.
4. **Later generative trials:** semantics, construction, ecology, aesthetics,
   assets and animation.
5. **Deferred runtime/network trials:** interest management and load-responsive
   partitioning only after runtime, multiplayer, consistency and threat-model
   decisions.

## Hybrid cadence and stop rules

- scheduled after three meaningful completed batches;
- early on verification failure, two repeated workarounds, milestone change,
  stagnation or a new input applicable to at least three systems;
- proposed probe cost is at most ten percent of the prior three batches and no
  more than one normal batch;
- local repository analysis precedes capped external research;
- one highest-value target runs at a time;
- Ordinary edits and heartbeat wakes never trigger divergence;
- no positive VOI, incomplete registry coverage, failed budget, fired
  falsifier or non-positive verified gain returns directly to mainline; and
- every result resumes the exact saved checkpoint.

## Explicit non-applications

This map does not authorize universal P2P, blockchain, erasure coding,
cryptocurrency, serverless MMO claims, fixed execution radii, global state
deletion, zero-byte universes, observer-dependent truth, global utility,
automatic research, implementation, approval, promotion, runtime selection,
spending, credentials, publishing or modification of completed systems.

## Mainline placement

The bounded framework implementation is an additive Forge improvement. After
its focused gates, mainline resumes at the revised fixed-160 interval-incident
implementation-readiness audit recorded under C3. The tangential audit cannot
replace or reorder that dependency route.
